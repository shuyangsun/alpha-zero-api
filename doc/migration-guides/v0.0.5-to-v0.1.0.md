# Migration Guide - v0.0.5 to v0.1.0

This is a breaking API redesign. The virtual `IGame` interface is gone, the
single `PolicyOutput` type is split, history is now engine-owned, and the
fixed-size policy layout is formalized through a per-game index map.

The TL;DR for porting a v0.0.5 game implementation:

1. Make your game class a value type that satisfies `concept Game`.
2. Replace `Apply(action)` with `ApplyActionInPlace(action)` and add
   `UndoLastAction()`. Never write your own `ApplyAction(game, action)`
   member — it now exists as a free function for any conforming `Game`.
3. Replace uses of `PolicyOutput` with `Evaluation` (network output) and
   `TrainingTarget` (z, π).
4. Update serializer/deserializer/augmenter signatures to take `const G&`
   (and, for the state serializer, a `RingBufferView<G>` history window).
5. Add the static contract `kHistoryLookback`, `kPolicySize`, `kMaxRounds`,
   plus a `PolicyIndex(action)` member.

## 1. `IGame` → `concept Game`

The virtual `IGame<B, A, P, E>` interface has been removed. Concrete games
are now plain value types satisfying `concept Game` (`game.h`); the engine is
templated on the concrete game type. This unblocks stack allocation, removes
virtual-dispatch overhead on the MCTS hot path, and lets the compiler see
`sizeof(G)`.

Before:

```cpp
using TttGameInterface = IGame<TttBoard, TttAction, TttPlayer, TttError>;
using TttGamePtr = std::unique_ptr<const TttGameInterface>;
using TttGameResult = TttResult<TttGamePtr>;

class TttGame final : public TttGameInterface {
 public:
  static TttGameResult Create();

  [[nodiscard]] TttGamePtr Apply(const TttAction& action) const noexcept final;
  // ... other observers ...
};

const TttGamePtr game = std::move(*TttGame::Create());
const TttGamePtr next = game->Apply(TttAction{1, 1});
```

After:

```cpp
class TttGame {
 public:
  using board_t = TttBoard;
  using action_t = TttAction;
  using player_t = TttPlayer;
  using error_t = TttError;

  static constexpr std::size_t kHistoryLookback = 0;
  static constexpr std::size_t kPolicySize = TTT_CELLS;
  static constexpr std::optional<uint32_t> kMaxRounds = 9;

  TttGame() = default;  // public; no factory required

  void ApplyActionInPlace(const TttAction& action) noexcept;
  void UndoLastAction() noexcept;

  [[nodiscard]] std::size_t PolicyIndex(const TttAction& a) const noexcept;
  // ... other observers ...
};

static_assert(::az::game::api::Game<TttGame>);

TttGame game;
game.ApplyActionInPlace(TttAction{1, 1});  // hot path: zero allocations
TttGame snapshot = ApplyAction(game, TttAction{0, 0});  // cold path: copy + apply
```

Notes:

- `Apply(action) -> unique_ptr<...>` is gone. The MCTS hot loop is expected
  to use `ApplyActionInPlace` / `UndoLastAction` on a single working copy;
  the free function `ApplyAction(game, action)` (defined once, in `game.h`)
  is for cold-path snapshots only. **Do not implement `ApplyAction` as a
  member** — it would shadow the free function.
- `kMaxRounds` is a self-play hard cap. If set, `IsOver()` must return true
  once `CurrentRound() >= *kMaxRounds`. Use `std::nullopt` for genuinely
  unbounded games.
- `error_t` carries over from v0.0.5; your existing `MyError` /
  `MyResult<T>` aliases keep working.
- `ITwoPlayer1DBoardGame` / `ITwoPlayer2DBoardGame` aliases in
  `defaults/game.h` have been removed — the concept replaces them. Keep
  using `Standard2DBoard`, `Action2D`, `BinaryPlayer`, etc.

## 2. `PolicyOutput` → `Evaluation` + `TrainingTarget`

`PolicyOutput` and its layout-baking constructors are gone. Network outputs
and training labels are now type-level distinct (`policy_output.h`):

```cpp
struct Evaluation {       // what the network produced
  float value;
  std::vector<float> probabilities;  // aligned to game.ValidActions()
};

struct TrainingTarget {   // what the network is asked to learn
  float z;
  std::vector<float> pi;             // aligned to game.ValidActions()
};
```

Deserializers return `Evaluation`. Policy-output serializers consume
`TrainingTarget`. Replay-buffer code that was passing `PolicyOutput` around
should switch to `TrainingTarget`; inference code should switch to
`Evaluation`.

In both types, the `i`-th probability/policy entry corresponds to
`game.ValidActions()[i]` — the same convention as before, just split.

## 3. Engine-owned history

The state serializer no longer takes raw board snapshots; it takes the
current game and a read-only window over recent past states owned by the
engine.

Before:

```cpp
class TttSerializer : public IGameSerializer<TttBoard, TttPlayer> {
  std::vector<float> SerializeCurrentState(
      const TttBoard& board, TttPlayer current) const noexcept final;
};
```

After:

```cpp
class TttSerializer : public IGameSerializer<TttGame> {
  std::vector<float> SerializeCurrentState(
      const TttGame& game,
      RingBufferView<TttGame> history) const noexcept final;
};
```

`history.Size()` is bounded by `G::kHistoryLookback`. Index 0 is the most
recent past state preceding `game`; `game` itself is **not** in the view.
Markov games declare `kHistoryLookback == 0` and always see an empty view —
the engine still owns the buffer; you just ignore it.

The engine side typically holds:

```cpp
using GameHistory =
    RingBuffer<TttGame, std::array<TttGame, TttGame::kHistoryLookback>>;
```

For `kHistoryLookback == 0`, `std::array<TttGame, 0>` occupies essentially
no space and `View()` always returns an empty view.

## 4. `PolicyIndex` and `kPolicySize`

The bijection between an `action_t` value and a slot in
`[0, kPolicySize)` is now part of the `Game` contract. The default
serializer/deserializer use it directly, which means they finally work
mid-game (not just at game start).

```cpp
// In your game class:
static constexpr std::size_t kPolicySize = TTT_CELLS;  // 9 for tic-tac-toe

[[nodiscard]] std::size_t PolicyIndex(const TttAction& a) const noexcept {
  return a.row * TTT_COLS + a.col;
}
```

The default policy serializer (`defaults/serializer.h`) writes
`out[1 + game.PolicyIndex(actions[i])] = target.pi[i]`, slot 0 holds
`target.z`. The default deserializer (`defaults/deserializer.h`) reads the
masked subset for `game.ValidActions()` and softmax-normalizes.

If you previously rolled your own policy serializer/deserializer just to
get masked indexing, you can probably delete it now and use the defaults.

The default deserializer hardcodes `std::string` as its error type. Games
that want a typed error should implement
`IPolicyOutputDeserializer<G, MyError>` directly.

## 5. Augmenter signatures

Both augmenters now exchange `G` instances rather than raw
`(board, player)` pairs.

```cpp
template <Game G>
class IInferenceAugmenter {
  std::vector<G> Augment(const G& game) const noexcept;
  Evaluation Interpret(
      const G& original,
      std::span<const G> augmented,
      std::span<const Evaluation> evaluations) const noexcept;
};

template <Game G>
class ITrainingAugmenter {
  std::vector<std::pair<G, TrainingTarget>> Augment(
      const G& game, const TrainingTarget& target) const noexcept;
};
```

Convention: `augmented[0]` is the identity. Games with no exploitable
symmetry can return a one-element vector; `Interpret` then becomes
effectively the identity.

If your augmenter applied a board symmetry, `Interpret` is responsible for
inverting that symmetry so the returned `Evaluation`'s probabilities align
with `original.ValidActions()`. The TtT augmenters in
`test/tic_tac_toe/inference.cc` and `train.cc` show one approach: keep an
`InverseTransformAction(action, sym)` helper, then re-key augmented action
indices through `game.PolicyIndex` to match the original ordering.

## 6. Quick checklist

- [ ] Game class is a value type; no virtual methods; no `unique_ptr` factory.
- [ ] `static_assert(Game<MyGame>)` compiles.
- [ ] `kHistoryLookback`, `kPolicySize`, `kMaxRounds` declared.
- [ ] `PolicyIndex(action)` defined.
- [ ] `ApplyActionInPlace` + `UndoLastAction` defined; allocation-free.
- [ ] No member `ApplyAction` — the free function in `game.h` covers it.
- [ ] Replay-buffer / inference code uses `TrainingTarget` / `Evaluation`.
- [ ] State serializer takes `RingBufferView<G>` history.
- [ ] Augmenters return `std::vector<G>` / `std::vector<std::pair<G, TrainingTarget>>`.
