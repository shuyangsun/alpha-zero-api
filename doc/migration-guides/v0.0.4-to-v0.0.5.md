# Migration Guide - v0.0.4 to v0.0.5

`IGame` now takes an explicit error type parameter for `ActionFromString()`.
For example, prefer absl-style naming like `TttError`, `TttResult<T>`, and
`TttStatus`.

Before:

```cpp
using TttGameInterface = IGame<TttBoard, TttAction, TttPlayer>;

[[nodiscard]] std::expected<TttAction, std::string> ActionFromString(
    std::string_view action_str) const noexcept final;
```

After:

```cpp
enum class TttError : uint8_t {
  kUnknownError = 0,
  kInvalidActionFormat,
};

template <typename T>
using TttResult = std::expected<T, TttError>;

using TttStatus = TttResult<void>;

using TttGameInterface = IGame<TttBoard, TttAction, TttPlayer, TttError>;

[[nodiscard]] TttResult<TttAction> ActionFromString(
    std::string_view action_str) const noexcept final;
```

Use `TttResult<TttGamePtr>` for constructors/factories and `TttStatus` for
status-only helpers. If you use the convenience aliases in `defaults/game.h`,
add the same error type there as well, for example
`ITwoPlayer2DBoardGame<ROWS, COLS, MyError>`.
