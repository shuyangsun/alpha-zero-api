# Migration Guide - v0.2.0 to v0.2.1

This release replaces the heap-allocating `Game::ValidActions()` contract
with caller-owned storage. The MCTS hot loop expands a node by calling the
game's legal-action enumerator, then immediately drops the result; under
the v0.2.0 contract that meant one `std::vector` allocation per expansion.
The new contract writes directly into a caller-supplied
`std::array<action_t, kMaxLegalActions>`, so the same MCTS loop allocates
zero per expansion.

`kMaxLegalActions` (added in v0.2.0) is now load-bearing — it sizes the
caller-owned buffer. The dense interfaces, compact interfaces, augmenter
interfaces, `Evaluation`, `TrainingTarget`, `RingBufferView<G>`, and
`PolicyIndex` are otherwise unchanged.

## What changed

`Game` no longer requires:

```cpp
std::vector<action_t> ValidActions() const noexcept;
```

It now requires:

```cpp
std::size_t ValidActionsInto(
    std::array<action_t, kMaxLegalActions>& out) const noexcept;
```

The method writes legal actions into `out[0..count)` and returns `count`.
Ordering must stay deterministic and state-only, just as `ValidActions()`
was. Callers must ignore entries at indices `>= count`. The ordering
contract is unchanged — a training tuple `(s, π, z)` written under one
ordering and replayed against a network trained under another is still
corrupt.

## What's unchanged

- `kHistoryLookback`, `kPolicySize`, `kMaxLegalActions`, `kMaxRounds`.
- `GetBoard` / `CurrentRound` / `CurrentPlayer` / `LastPlayer` /
  `LastAction` / `CanonicalBoard` / `IsOver` / `GetScore` / `PolicyIndex`.
- `ApplyActionInPlace` / `UndoLastAction` and the free
  `ApplyAction(game, action)` cold-path helper.
- `Evaluation`, `TrainingTarget`, `CompactPolicyTargetBlob`,
  `CompactPolicyOutputBlob`, and the four serializer/deserializer
  interfaces.
- `RingBufferView<G>` and the engine-owned history contract.
- The augmenter interfaces themselves — they exchange `G` instances and
  `TrainingTarget`s, both layout-agnostic, so the abstract method
  signatures do not change. The concrete augmenter implementations that
  *read* legal actions inside `Interpret` / `Augment` do need to swap
  `ValidActions()` for `ValidActionsInto(...)`; see the TTT augmenters
  under `test/tic_tac_toe/` for the pattern.

## Game migration

Before:

```cpp
std::vector<Action> MyGame::ValidActions() const noexcept {
  std::vector<Action> actions;
  actions.reserve(kMaxLegalActions);
  for (Action action : AllCandidateActions()) {
    if (IsLegal(action)) {
      actions.push_back(action);
    }
  }
  return actions;
}
```

After:

```cpp
std::size_t MyGame::ValidActionsInto(
    std::array<Action, kMaxLegalActions>& out) const noexcept {
  std::size_t count = 0;
  for (Action action : AllCandidateActions()) {
    if (IsLegal(action)) {
      out[count] = action;
      ++count;
    }
  }
  return count;
}
```

`<vector>` may no longer be needed in `game.h` if `ValidActions` was its
only user; replace the include with `<array>` and `<cstddef>`.

## Caller migration

Before:

```cpp
const std::vector<Action> actions = game.ValidActions();
for (std::size_t i = 0; i < actions.size(); ++i) {
  Use(actions[i]);
}
```

After:

```cpp
std::array<Action, MyGame::kMaxLegalActions> actions{};
const std::size_t count = game.ValidActionsInto(actions);
for (std::size_t i = 0; i < count; ++i) {
  Use(actions[i]);
}
```

The buffer is stack-allocated; cost is `sizeof(action_t) *
kMaxLegalActions` per call site, paid by the caller's frame, not the
heap.

## Defaults and augmenters

The dense and compact default implementations
(`defaults/{compact_,}serializer.h`, `defaults/{compact_,}deserializer.h`)
have been updated internally to use `ValidActionsInto`. Games using only
the defaults need **no source change** in their serializer / deserializer
classes — just bump the API pin and re-implement
`Game::ValidActionsInto`.

Custom serializers, custom deserializers, and custom augmenters that
called `game.ValidActions()` need the caller-migration pattern above.
The TTT example under `test/tic_tac_toe/` shows the pattern for a custom
deserializer (`deserializer.cc`), an inference augmenter (`inference.cc`),
and a training augmenter (`train.cc`).

## Notes and gotchas

- The buffer length is `G::kMaxLegalActions`, not `kPolicySize`. If you
  previously sized scratch buffers to `kPolicySize`, that still works
  but wastes stack on compact games. Prefer `kMaxLegalActions` going
  forward.
- The returned `count` is the only authority for "how many entries are
  valid in `out`." Do not call `.size()` on the buffer or iterate past
  `count`; entries `>= count` are unspecified.
- If your `ValidActions()` returned an empty vector when the game was
  over (or never), `ValidActionsInto` should return `0` for the same
  states — leave `out` untouched and return `0`.
- `kMaxLegalActions == 0` is a configuration error: the new concept
  needs a non-zero buffer size for any state with legal moves. Games
  generated with `policy_head_layout=compact` in the cookiecutter
  template still carry a `TODO(policy_head_layout=compact)` marker on
  the `kMaxLegalActions` declaration — fill it in before implementing
  `ValidActionsInto`.

## Checklist

- [ ] Remove `ValidActions()` from the game class.
- [ ] Add `ValidActionsInto(std::array<action_t, kMaxLegalActions>&)`
      returning the legal-action count.
- [ ] Keep the action order identical to the previous `ValidActions()`
      order.
- [ ] `static_assert(::az::game::api::Game<MyGame>)` still compiles.
- [ ] Replace caller-side `.size()` checks with the returned count;
      replace `const auto actions = game.ValidActions();` with the
      `std::array` + `ValidActionsInto` pair.
- [ ] Update custom serializers / deserializers / augmenters that read
      legal actions. Games using only the defaults need no change in
      those classes.
- [ ] `<vector>` removed from `game.h` if it was only there for
      `ValidActions`; `<array>` / `<cstddef>` added if missing.
- [ ] Build configures cleanly against `alpha-zero-api@v0.2.1` (check
      `configuration.h` reports `_PATCH == 1`).
- [ ] Game-specific tests still pass.
