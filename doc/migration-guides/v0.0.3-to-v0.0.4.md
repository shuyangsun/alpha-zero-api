# Migration Guide - v0.0.3 to v0.0.4

Added `[[nodiscard]]` and `noexcept` to all public interface methods whereever
applicable. Exceptions are forbidden in any game implementation.

Also changed the example test implementation's native game constructor to be
private. Game creation should be done with public static methods that returns
result types. Example:

```cpp
namespace {

enum class TttError : uint8_t {
  kUnknownError = 0,
  kNotImplemented,
};

using TttGamePtr = std::unique_ptr<const TttGameInterface>;
using TttGameResult = std::expected<TttGamePtr, TttError>;

// ... other code

}  // namespace

TttGameResult game_result = TttGame::Create();
if (!game_result.has_value()) {
  std::cerr << "Error creating game: "
            << static_cast<std::underlying_type_t<TttError>>(
                   game_result.error())
            << std::endl;
  return 1;
}
const TttGamePtr game = std::move(*game_result);
```
