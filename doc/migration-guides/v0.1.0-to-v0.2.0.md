# Migration Guide - v0.1.0 to v0.2.0

This release adds a single static-member contract to the `Game` concept and
ships a parallel set of compact policy-output interfaces. The dense
interfaces and their defaults are unchanged. The only breaking change is the
new `kMaxLegalActions` static — every existing game must declare it.

## Quick path for dense games

If you are happy with the dense `[z, p_0, ..., p_{kPolicySize-1}]` policy
output (the canonical AlphaZero layout), the entire migration is one line in
your game class:

```cpp
static constexpr std::size_t kMaxLegalActions = kPolicySize;
```

`static_assert(::az::game::api::Game<MyGame>)` should then keep compiling.
No serializer, deserializer, augmenter, or network change is needed.

## What's new in the API

| Symbol | Lives in | Role |
| --- | --- | --- |
| `G::kMaxLegalActions` (concept addition) | `game.h` | Per-state legal-action ceiling. Must satisfy `kMaxLegalActions <= kPolicySize`. |
| `CompactPolicyTargetBlob` | `policy_output.h` | Replay-buffer label for a compact policy head. |
| `CompactPolicyOutputBlob` | `policy_output.h` | Network-output read-side of the same layout. |
| `ICompactPolicyOutputSerializer<G>` | `serializer.h` | Compact analogue of `IPolicyOutputSerializer<G>`. |
| `ICompactPolicyOutputDeserializer<G, E>` | `deserializer.h` | Compact analogue of `IPolicyOutputDeserializer<G, E>`. |
| `DefaultCompactPolicyOutputSerializer<G>` | `defaults/compact_serializer.h` | Canonical compact scatter from `TrainingTarget`. |
| `DefaultCompactPolicyOutputDeserializer<G>` | `defaults/compact_deserializer.h` | Canonical compact gather into `Evaluation`. |

`Evaluation`, `TrainingTarget`, `RingBufferView<G>`, the augmenter
interfaces, and the dense interface are **unchanged**.

## Switching to a compact policy head

The compact path is for games where the per-state legal-action ceiling is
orders of magnitude smaller than the full action space. Read the
`designing-serialization` SKILL's "Dense vs compact policy head" section
before you decide; the rule of thumb is `kPolicySize > 10⁴` *and*
`kMaxLegalActions / kPolicySize < 0.1`.

If you commit to compact:

1. Pick `kMaxLegalActions` — the tightest per-state upper bound on
   `|ValidActions()|` you can prove. It only needs to be tight enough that
   the network's compact head doesn't waste rows.
2. Replace your `IPolicyOutputSerializer<G>` implementation with one that
   conforms to `ICompactPolicyOutputSerializer<G>`. The default is
   `DefaultCompactPolicyOutputSerializer<G>`.
3. Replace your `IPolicyOutputDeserializer<G, E>` implementation with one
   that conforms to `ICompactPolicyOutputDeserializer<G, E>`. The default is
   `DefaultCompactPolicyOutputDeserializer<G>` (which uses `std::string`).
4. Resize the network's policy-head output projection to width
   `kMaxLegalActions` (typically a pointer-network attending over per-action
   input tokens). Update your training pipeline / replay-buffer schema to
   match the compact blob.

## Before / after — dense game (TTT)

Before:

```cpp
class TttGame {
 public:
  static constexpr std::size_t kHistoryLookback = 0;
  static constexpr std::size_t kPolicySize = 9;
  static constexpr std::optional<uint32_t> kMaxRounds = 9;
  // ...
};
```

After:

```cpp
class TttGame {
 public:
  static constexpr std::size_t kHistoryLookback = 0;
  static constexpr std::size_t kPolicySize = 9;
  static constexpr std::size_t kMaxLegalActions = kPolicySize;  // dense
  static constexpr std::optional<uint32_t> kMaxRounds = 9;
  // ...
};
```

The serializer, deserializer, augmenters, and network are untouched.

## Before / after — switching to compact

For a hypothetical card game switching to compact, the diff lives at the
interface boundary:

```cpp
// Before — dense
class CardGame { /* ... */
  static constexpr std::size_t kPolicySize = 250'000;
  // (no kMaxLegalActions in v0.1.0)
};

class CardSerializer : public IPolicyOutputSerializer<CardGame> {
  std::vector<float> SerializePolicyOutput(
      const CardGame& g, const TrainingTarget& t) const noexcept final;
};

class CardDeserializer
    : public IPolicyOutputDeserializer<CardGame, CardError> {
  std::expected<Evaluation, CardError> Deserialize(
      const CardGame& g, std::span<const float> out) const noexcept final;
};
```

```cpp
// After — compact (v0.2.0)
class CardGame { /* ... */
  static constexpr std::size_t kPolicySize = 250'000;
  static constexpr std::size_t kMaxLegalActions = 12;  // hand-size ceiling
};

class CardSerializer : public ICompactPolicyOutputSerializer<CardGame> {
  CompactPolicyTargetBlob SerializePolicyOutput(
      const CardGame& g, const TrainingTarget& t) const noexcept final;
};

class CardDeserializer
    : public ICompactPolicyOutputDeserializer<CardGame, CardError> {
  std::expected<Evaluation, CardError> Deserialize(
      const CardGame& g, const CompactPolicyOutputBlob& out) const noexcept final;
};
```

The network's policy-head output projection drops from
`hidden × 250'000` to `hidden × 12`, and the replay-buffer row stores a
`CompactPolicyTargetBlob` (one value scalar + 12 indices + 12 values)
instead of `1 + 250'000` floats per sample.

## Notes and gotchas

- `kMaxLegalActions <= kPolicySize` is required.
  `DefaultPolicyOutputSerializer<G>` `static_asserts` it; if you write a
  custom dense serializer, assert it yourself.
- The compact deserializer expects every `ValidActions()` slot to appear in
  the network output's `legal_indices`. Networks that emit a fixed-width
  row may need to pad shorter rows with
  `CompactPolicyTargetBlob::kPaddingSlot`; the deserializer ignores
  padding entries.
- Compact and dense interfaces are **not interchangeable**. A game picks
  one for the entire pipeline (serializer, deserializer, network head).
  Mixing them is a configuration error and the default deserializers
  assert the size assumptions of their layout.
- `RingBufferView<G>`, `Evaluation`, `TrainingTarget`, and the augmenter
  interfaces are unchanged. Augmenters exchange `G` instances and
  `TrainingTarget`s — both layout-agnostic — so no augmenter change is
  needed when switching to compact.

## Quick checklist

- [ ] `static constexpr std::size_t kMaxLegalActions` declared on the
      game class; satisfies `kMaxLegalActions <= kPolicySize`.
- [ ] `static_assert(::az::game::api::Game<MyGame>)` still compiles.
- [ ] If staying dense: no other code changes needed.
- [ ] If switching to compact: serializer + deserializer + network
      head swapped to the compact triple; downstream training
      pipeline updated.
- [ ] Build configures cleanly against `alpha-zero-api@v0.2.0` (check
      `configuration.h` reports `_MINOR == 2`).
- [ ] Game-specific tests still pass.
